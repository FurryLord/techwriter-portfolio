## Быстрый старт

### Требования
- [Hugo](https://gohugo.io/getting-started/installing/) (рекомендуется extended версия)
- Git

### Установка зависимостей

```bash
# Клонирование репозитория (если ещё не клонирован)
git clone <your-repo-url>
cd techwriter-portfolio

# Инициализация git submodules (тема hugo-book)
git submodule init
git submodule update
```

### Запуск сервера разработки

```bash
hugo server
```

Сайт будет доступен по адресу: http://localhost:1313

### Сборка для продакшена

```bash
hugo --minify
```

Готовые файлы будут в папке `public/`.

## Настройка

### Основные параметры

Отредактируйте `hugo.toml`:

```toml
baseURL = 'https://your-domain.com/'
title = 'Ваше имя - Технический писатель'
languageCode = 'ru-ru'

[params]
  BookTheme = 'light'  # или 'dark', или 'auto'
  BookSearch = true    # Включить поиск
```

### Добавление контента

Создайте новую страницу:

```bash
hugo new docs/your-page.md
```

Или создайте файл вручную в папке `content/docs/`.

### Структура меню

Меню настраивается в `hugo.toml` или через front matter страниц:

```yaml
---
title: "Название страницы"
weight: 10  # Порядок в меню
---
```

## Развёртывание

### GitHub Pages (Рекомендуемый способ)

Проект настроен для автоматического развертывания на GitHub Pages с помощью GitHub Actions.

#### Настройка:

1. **Создайте репозиторий на GitHub**
   ```
   Имя репозитория: techwriter-portfolio
   Тип: Public
   ```

2. **Загрузите код в репозиторий**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/techwriter-portfolio.git
   git push -u origin main
   ```

3. **Включите GitHub Pages**
   - Перейдите в Settings → Pages
   - Source: "GitHub Actions"
   - Сайт будет доступен по адресу: `https://YOUR_USERNAME.github.io/techwriter-portfolio/`

4. **Автоматическое развертывание**
   - При каждом пуше в ветку `main` запускается CI/CD пайплайн
   - Проверяется качество текста и кода
   - Сайт автоматически собирается и публикуется

#### CI/CD Проверки:

- ✅ **Проверка орфографии** (русский и английский)
- ✅ **Качество текста** (грамматика, стиль, терминология)
- ✅ **Проверка ссылок** (внутренние и внешние)
- ✅ **Markdown форматирование**
- ✅ **HTML валидация**
- ✅ **Hugo сборка**

### Netlify (Альтернативный способ)

1. Подключите репозиторий к Netlify
2. Установите команду сборки: `hugo --minify`
3. Установите папку публикации: `public`

## Кастомизация

### Цветовая схема

Измените в `hugo.toml`:

```toml
[params]
  BookTheme = 'dark'  # light, dark, auto
```

### Логотип

Добавьте логотип в `static/logo.png` и настройте:

```toml
[params]
  BookLogo = 'logo.png'
```

### Кастомные стили

Создайте файл `assets/_custom.scss` с вашими стилями.
